SECTION 2: EVIDENCE ADMISSIBILITY ASSESSMENT
================================================================================

2.1 DAUBERT STANDARD (FEDERAL COURT)

The Daubert standard (Daubert v. Merrell Dow Pharmaceuticals, 509 U.S. 579,
1993) requires expert testimony to be based on scientifically valid methodology.
The court evaluates four factors:

FACTOR 1: TESTABILITY
Can the methodology be tested and has it been tested?

ASSESSMENT: YES - STRONG
- Build version extraction methodology tested on 500+ file dataset
- Cross-verification with multiple independent tools (ExifTool, hex editors)
- Reproducible by any qualified examiner using same tools
- Error rate: 0.2% on metadata extraction (validated)

FACTOR 2: PEER REVIEW AND PUBLICATION
Has the methodology been subject to peer review and publication?

ASSESSMENT: MODERATE
- DWG/RVT file forensics not widely published (emerging field)
- Underlying techniques (binary file analysis, metadata extraction) well-
  established in digital forensics literature
- NIST 800-86 and ISO 27037 standards peer-reviewed and widely accepted
- LibreDWG open-source parser (peer-reviewed) used for validation

MITIGATING STRATEGY:
Emphasize that specific file format is new, but methodology (binary analysis,
hash validation, metadata extraction) is established forensic practice used in
thousands of cases.

FACTOR 3: ERROR RATE
What is the known or potential error rate?

ASSESSMENT: YES - STRONG
- DWG Forensic Tool: 0.2% error rate on metadata extraction (500-file validation)
- Build version extraction: 0% error rate (deterministic, not probabilistic)
- Cross-verification eliminates tool-specific errors
- Manual hex analysis provides independent confirmation

FACTOR 4: GENERAL ACCEPTANCE
Is the methodology generally accepted in the relevant scientific community?

ASSESSMENT: YES - MODERATE-STRONG
- Binary file analysis: Universally accepted in digital forensics
- Metadata extraction: Standard practice (used by law enforcement, FBI)
- CAD file forensics: Emerging but growing acceptance
- Expert qualified in multiple prior cases

OVERALL DAUBERT ASSESSMENT: HIGH PROBABILITY OF ADMISSION

The methodology satisfies all four Daubert factors. Expect successful
qualification. Opposing counsel may challenge Factor 2 (peer review) due to
CAD forensics being specialized, but factors 1, 3, and 4 are strong.

DAUBERT HEARING STRATEGY:
If opposing counsel requests Daubert hearing, emphasize:
- Methodology follows established NIST/ISO standards
- Findings independently verifiable (any examiner can confirm build versions)
- Error rate low and quantified
- Expert qualified in multiple prior cases

2.2 FRYE STANDARD (STATE COURT ALTERNATIVE)

The Frye standard (Frye v. United States, 293 F. 1013, 1923) requires
methodology to be "generally accepted" in the relevant scientific community.

ASSESSMENT: MODERATE-HIGH PROBABILITY OF ADMISSION

CAD file forensics is emerging, but underlying techniques (digital forensics,
binary analysis, metadata validation) are universally accepted. Emphasize:
- Expert follows same methodology as FBI digital forensics
- Tools and techniques used in thousands of criminal and civil cases
- Autodesk file format specifications publicly available
- Findings reproducible by any qualified examiner

FRYE JURISDICTIONS STRATEGY:
Focus on general acceptance of digital forensics, not specific file format.
Argument: "The file format is new, but the forensic methodology is established
and accepted for decades."

3.1 ANTICIPATED DEFENSE EXPERT STRATEGY

Based on the strength of forensic evidence, opposing expert will likely employ
one of three strategies:

STRATEGY A: CHALLENGE METHODOLOGY
Attack the reliability of custom forensic tools, question validation, suggest
alternative interpretations of build versions.

VULNERABILITY:
Build version is directly readable in file structure. Any expert can verify it
with a hex editor. Challenging the methodology requires explaining why multiple
independent tools (DWG Forensic Tool, ExifTool, hex editor) all show identical
results.

COUNTER-STRATEGY:
Offer live demonstration in court. Open files in hex editor, show build version
at offset 0x0B. Ask opposing expert: "Do you dispute this is the build version?"

STRATEGY B: PROPOSE INNOCENT EXPLANATIONS
Suggest software bugs, automatic Revit processes, or user errors that could
account for build version discrepancy.

VULNERABILITY:
Any innocent explanation must account for ALL THREE forensic anomalies:
1. Chronological impossibility (backup newer than primary)
2. Deleted partition containing detailed BBQ amenities
3. Violation of Revit backup naming conventions

Defense expert must explain how all three occurred together by innocent means.
This is logically untenable.

COUNTER-STRATEGY:
During cross-examination of defense expert, ask: "Can you provide a single
explanation that accounts for the build version discrepancy, the deleted
amenities, and the backup naming violation?" If they cannot, their testimony
lacks coherence.

STRATEGY C: MINIMIZE SIGNIFICANCE
Argue that build version differences are "not important" or "common" or "don't
prove manipulation."

VULNERABILITY:
Build versions are fundamental to software versioning and are used by Autodesk
for support, bug tracking, and compatibility. Suggesting they're meaningless
contradicts Autodesk's own engineering practices.

COUNTER-STRATEGY:
Call Autodesk technical expert as rebuttal witness to testify that build
versions are reliable, automatically assigned, and cannot be manually altered.
This neutralizes any attempt to minimize significance.

3.2 SPECIFIC VULNERABILITIES TO EXPLOIT

VULNERABILITY 1: LOGICAL IMPOSSIBILITY

The chronological impossibility creates an inescapable logical trap:
- If Lane.0024 is a legitimate backup, it cannot have a newer build version
- If Lane.0024 was created later, it cannot be a backup
- There is no third option

Any defense expert attempting to explain this must violate basic logic. Use
this to create credibility doubt.

CROSS-EXAMINATION SCRIPT:
Q: "Dr. [Defense Expert], you agree that Revit build 20210224 was released
    February 24, 2021?"
A: [Must agree - objectively true]

Q: "And you agree that Revit build 20210921 was released September 21, 2021?"
A: [Must agree - objectively true]

Q: "So Lane.0024, with build 20210921, was created with software released 7
    months AFTER the software used for Lane.rvt?"
A: [Must agree or lose credibility]

Q: "Can you explain how a backup file created to preserve an earlier design
    state was created with software that didn't exist when the primary file
    was made?"
A: [No coherent answer possible]

VULNERABILITY 2: BURDEN OF PROOF ON ALTERNATIVE EXPLANATION

If defense expert proposes an innocent explanation (e.g., software bug, user
error), they bear burden of proving it's PLAUSIBLE given all evidence.

CROSS-EXAMINATION SCRIPT:
Q: "Dr. [Defense Expert], you're suggesting this was caused by [proposed
    explanation]?"
A: "Yes."

Q: "Has this ever been reported to Autodesk as a bug?"
A: [Likely "No" - no known bug reports]

Q: "Have you been able to reproduce this behavior in testing?"
A: [Likely "No" - behavior is not reproducible]

Q: "Can you point to any documentation from Autodesk confirming this behavior?"
A: [Likely "No"]

Q: "So you're asking the jury to accept an explanation that has never been
    documented, never been reproduced, and never been reported by any other
    Revit user in the software's 20-year history?"
A: [Credibility damaged]

VULNERABILITY 3: DELETED PARTITION EVIDENCE

If defense expert focuses solely on build versions and ignores deleted partition,
exploit this omission.

CROSS-EXAMINATION SCRIPT:
Q: "Dr. [Defense Expert], you're aware that a 3.2-megabyte partition was deleted
    in Lane.0024 but is active in Lane.rvt?"
A: [If "No" - expert didn't conduct thorough analysis. If "Yes" - proceed]

Q: "You're aware this deleted partition contains 17 instances of 'BBQ' with
    detailed amenity specifications?"
A: [If "No" - incomplete analysis. If "Yes" - proceed]

Q: "Do you have an innocent explanation for why amenities would be systematically
    deleted from a file at the same time it was given a misleading backup name?"
A: [No coherent innocent explanation exists]

VULNERABILITY 4: FINANCIAL BIAS

Defense expert is also being paid. If they attack our expert's compensation,
turn the attack around.

CROSS-EXAMINATION SCRIPT:
Q: "Dr. [Defense Expert], how much are you being paid for your testimony today?"
A: [Must disclose or appear evasive]

Q: "Is your compensation contingent on your conclusions?"
A: [Must say "No" or lose credibility]

Q: "So you're being paid the same amount whether you find manipulation or not?"
A: [Must agree]

Q: "Just like [plaintiff's expert]?"
A: [Neutralizes bias attack]

WHAT THEY ARE:
Revit maintains transaction log files (journal files) that record every action

WHY CRITICAL:

DISCOVERY REQUEST LANGUAGE:

If files were stored in Dropbox (or other cloud storage), Dropbox maintains

4.3 WORKSTATION FORENSIC IMAGE

WHAT IT IS:
Bit-for-bit copy of the hard drive from the workstation used to create the
Revit files, preserving all data including deleted files, temporary files,
system logs, and metadata.

WHY CRITICAL:
Workstation forensic image will reveal:
- File system timestamps (creation, modification, access)
- Deleted file recovery (intermediate versions, original files before manipulation)
- Windows Event Logs (Revit application launches, file operations)
- Browser history (did user research "how to backdate files" or similar?)
- Email/chat deleted items (communications about file manipulation)

DISCOVERY REQUEST LANGUAGE:
"Defendant shall preserve and produce a forensically sound image of the complete
hard drive or solid-state drive of any workstation computer used to create,
modify, or store the files 'Lane.rvt' or 'Lane.0024', including all operating
system files, application files, user data, deleted files, and unallocated space.
Imaging shall be performed by a qualified forensic examiner using write-blocking
technology in accordance with NIST SP 800-86."

EXPECTED DEFENDANT RESISTANCE:
Defendant will likely object to this as overly broad, burdensome, and invading
privacy/privilege. Negotiate for:
- Court-appointed neutral forensic examiner
- Privilege screening protocol
- Limited scope (specific date range, specific file types)

COST-BENEFIT ANALYSIS:
Workstation imaging is expensive (forensic examiner fees, attorney review time).
Only pursue if:
- Revit journal files not available (spoliation suspected)
- Case value justifies expense
- Strong indication additional evidence exists on workstation

SECTION 6: RECOMMENDED JURISDICTION
================================================================================

Based on admissibility considerations and strategic advantages, federal court
is strongly preferred over state court for this litigation.

6.1 FEDERAL COURT ADVANTAGES

DAUBERT STANDARD:
Federal courts apply Daubert v. Merrell Dow Pharmaceuticals (509 U.S. 579, 1993),
which is more flexible for emerging forensic techniques. Daubert focuses on
methodology reliability rather than general acceptance, favoring our scientifically
sound analysis even if CAD forensics is specialized.

JUDICIAL EXPERIENCE:
Federal judges have greater experience with digital forensics expert testimony,
particularly in intellectual property and fraud cases. Less time required to
educate judge on technical concepts.

TECHNOLOGY RESOURCES:
Federal courtrooms typically have superior technology infrastructure for
presenting digital evidence (large displays, reliable connectivity, technical
support).

EXPERT WITNESS STANDARDS:
Federal courts are accustomed to complex expert testimony and less likely to
exclude expert evidence based on novelty of file format.

6.2 STATE COURT DISADVANTAGES (FRYE JURISDICTIONS)

GENERAL ACCEPTANCE STANDARD:
States applying Frye v. United States (293 F. 1013, 1923) require methodology
to be "generally accepted" in scientific community. CAD file forensics being
emerging field creates admission risk in Frye jurisdictions.

FRYE STATES TO AVOID:
- California (Frye-plus standard, Kelly factors)
- Florida (Frye standard)
- Illinois (Frye standard)
- Maryland (Frye-Reed standard)
- New York (Frye standard)

SECTION 7: SPOLIATION ARGUMENTS
================================================================================

If discovery reveals that defendant failed to preserve relevant evidence
(Revit journal files, intermediate versions, communications), pursue spoliation
sanctions.

7.1 DUTY TO PRESERVE

Duty to preserve evidence arises when litigation is reasonably foreseeable.
Establish:
- When did amenity disputes arise? (trigger for preservation duty)
- When did defendant become aware of potential litigation?
- What evidence existed at that time?
- What evidence was destroyed or allowed to be auto-deleted?

7.2 SPOLIATION REMEDIES

Depending on severity and intent, request:

MILD SANCTIONS (Negligent Spoliation):
- Adverse inference instruction: Jury instructed to infer destroyed evidence
  would have been unfavorable to defendant
- Cost-shifting: Defendant pays plaintiff's additional discovery costs

SEVERE SANCTIONS (Intentional Spoliation):
- Default judgment or dismissal of defenses
- Preclusion of defendant's expert testimony
- Monetary sanctions

7.3 SPOLIATION EVIDENCE TO DEVELOP

If defendant claims Revit journal files "no longer exist":
- When were they deleted?
- Was deletion automatic or manual?
- Did defendant have IT policy to preserve litigation-relevant data?
- Was defendant aware journal files could contain relevant evidence?

If defendant "no longer has" intermediate file versions:
- Were files stored in cloud with version history?
- When was version history purged?
- Did defendant take affirmative steps to delete versions?
